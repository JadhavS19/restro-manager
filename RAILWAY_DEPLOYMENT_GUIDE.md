# 🚀 RestroManager: Complete Railway Deployment Guide

This guide provides a step-by-step walkthrough to deploy your Restaurant Management System (Backend + Frontend + MySQL) to Railway.

---

## 📋 Prerequisites
- A GitHub repository with your current code (Done! It's already pushed).
- A [Railway.app](https://railway.app) account.

---

## 🗄️ Step 1: Provision the MySQL Database
1.  Log in to **Railway**.
2.  Click **"+ New"** (big purple button).
3.  Choose **"Database"** -> **"Add MySQL"**.
4.  Wait for the status to change from "Provisioning" to **"Active"**.
5.  *(Optional)* Click on the MySQL service and go to the **Data** tab to see your tables (they will be empty for now).

---

## ⚙️ Step 2: Deploy the Backend
1.  In your Railway project, click **"+ New"** -> **"GitHub Repo"**.
2.  Select your repository (`restro-manager`).
3.  Choose **"backend"** as the directory to deploy.
4.  **Important:** Go to the **Variables** tab of the Backend service and add these:
    - **`MYSQL_URL`**: Click "Add Variable" -> Source: `${{MySQL.MYSQL_URL}}`.
    - **`JWT_SECRET`**: Add a random strong password (e.g., `restro_master_key_2024`).
    - **`PORT`**: `5000`
5.  Go to **Settings** -> **Public Networking** -> click **"Generate Domain"**.
6.  **Copy this URL** (it should look like `https://backend-production-xyz.up.railway.app`).

---

## 🎨 Step 3: Deploy the Frontend
1.  In the same Railway project, click **"+ New"** -> **"GitHub Repo"**.
2.  Select your repository again (`restro-manager`).
3.  Choose **"frontend/restaurant-management-sytem lov"** as the directory.
4.  Go to the **Variables** tab and add:
    - **`VITE_API_URL`**: Paste your Backend URL + `/api`.
      *(e.g., `https://backend-production-xyz.up.railway.app/api`)*.
5.  Go to **Settings** -> **Public Networking** -> click **"Generate Domain"**.
6.  **This is your final live website link!**

---

## 🌱 Step 4: Initial Data (Seeding)
Your database is currently empty. To add the admin user and the 30+ menu items:
1.  Click on your **Backend** service in Railway.
2.  Go to the **Settings** tab.
3.  Find the **"Build & Deploy"** section.
4.  Look for a **"Web Console"** or **"Shell"** icon (or use the Railway CLI if you have it).
5.  **Run this command** in the terminal:
    ```bash
    npm run seed
    ```
6.  Alternatively, you can temporarily change your "Start Command" in Railway Settings to `npm run seed`, let it run once, and change it back to `npm start`.

## 🛠️ Troubleshooting
- **Database Connection Error?** Check if `MYSQL_URL` is correctly linked in the Backend variables.
- **Frontend can't talk to Backend?** Ensure `VITE_API_URL` has `https://` at the start and `/api` at the end.
- **Tables not showing?** Restart the Backend service once. Sequelize will create them on the first successful start.
